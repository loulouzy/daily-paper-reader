## 摘要
确保大型语言模型（LLM）在现实部署中的安全性和上下文适当行为仍然是一个关键挑战。我们提出了SafeCtrl-RL，一个推理时行为控制框架，能够在不进行模型重训练或参数修改的情况下实现自适应安全调节。该方法将对话生成建模为顺序决策过程，其中强化学习代理根据上下文反馈动态选择提示调整策略。这样，不安全行为可以通过迭代精炼被抑制，我们将此概念化为推理时行为忘却。在多个LLM和不安全对话场景上的评估表明，SafeCtrl-RL持续提高了安全性和响应质量，优于现有的基于提示的优化方法，并在性能与效率之间取得了良好的平衡。**警告：本文可能包含有害语言的示例，建议读者谨慎阅读。

## Abstract
Ensuring safe and contextually appropriate behaviour in Large Language Models (LLMs) remains a critical challenge for real-world deployment. We present \textbf{SafeCtrl-RL}, an inference-time behavioural control framework that enables adaptive safety regulation without model retraining or parameter modification. The method formulates dialogue generation as a sequential decision process, where a reinforcement learning agent dynamically selects prompt adjustment strategies based on contextual feedback. This allows unsafe behaviours to be suppressed through iterative refinement, which we conceptualise as inference-time behavioural unlearning. Evaluated across multiple LLMs and unsafe dialogue scenarios, SafeCtrl-RL consistently improves safety and response quality, outperforms existing prompt-based optimisation methods, and achieves favourable performance--efficiency trade-offs. **Warning: This paper may contain examples of harmful language, and reader discretion is recommended.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：大型语言模型（LLM）在交互式对话场景中，确保安全且上下文恰当的行为仍是实际部署的重大挑战。现有方法（如微调、模型编辑）需要重训练或修改参数，成本高，且无法在推理时动态适应对话上下文的变化。基于静态提示的方法缺乏自适应能力。
- **整体含义**：本文提出一种推理时行为控制框架 **SafeCtrl-RL**，通过强化学习（RL）代理动态选择提示调整策略，在**不修改模型参数、不重新训练**的前提下，迭代抑制不安全输出，实现“推理时行为忘却”（inference-time behavioural unlearning）。

## 2. 论文提出的方法论：核心思想、关键技术细节与公式/算法流程

- **核心思想**：将对话生成建模为一个**封闭循环的生成-评估-精炼**过程，其中RL代理根据当前对话状态（包含历史反馈、性能指标等）从预定义的11种策略空间中选择最优策略来更新系统提示，引导LLM生成更安全且质量更高的回复。
- **关键技术细节**：
  - **任务形式化**：给定固定参数LLM M和水话轮t、精炼轮次k，用户输入U_t，系统提示S_t^{(k)}，生成响应R_t^{(k)} = M(U_t, S_t^{(k)})。经安全-质量评估器打分，若不满足阈值则更新提示。
  - **安全-质量评估器**：使用DeepEval框架，结合质量分数（连贯性、相关性）和安全分数（毒性、仇恨、暴力等12类危害），最终得到联合标量得分作为奖励信号。
  - **状态空间**：36维向量，包含元学习特征（精炼进度、探索率）、分数特征（最近性能统计、策略历史有效性）、提示特征（危害类别、风险等级、哈希指纹）。
  - **动作空间**：11种离散策略（如最小历史、原始历史、AI摘要、渐进摘要、对比学习、自适应等）。
  - **奖励函数**：指数加权乘积 \( r(q,s) = q^{\alpha\beta} s^{(1-\alpha)\beta} \)，其中\(\alpha=0.6\)控制质量-安全权衡，\(\beta=10.0\)控制奖励曲面陡峭度。另设硬安全阈值，违反则奖励为0。
  - **算法流程**：采用DQN训练RL代理，状态编码后选择动作更新提示，生成新响应，经评估后获得奖励，存回经验池，迭代直至满足停止准则或达到最大轮次。

## 6. 论文的主要结论与发现

- SafeCtrl-RL 在**所有模型**上一致取得最高平均 Macro-PScore（0.818），显著优于所有手工策略（最高 0.807）和提示优化方法（最高 0.342）。
- 行为改善显著（ΔP 0.360~0.641），且**性能-效率比最优**（更高的 ΔP 与更少的迭代次数 n ≈ 3.7）。
- 保留分析表明，移除保护后性能虽有下降，但仍明显高于基线，说明部分改善得以保留（最高 Evil-Alpaca Retention 94.5%，但 DialoGPT 仅 60.1%），**保留程度依赖于模型内在对齐**。
- 现有提示优化方法（如 OPRO、Evolutionary）在安全关键场景下常出现负面或极小的改善，表明它们不适合在线安全控制。

## 7. 优点：方法或实验设计上的亮点

- **新颖性**：将对话安全控制形式化为推理时序列决策问题，结合 RL 实现动态、自适应的提示调整策略，无需访问模型参数或重训练。
- **通用性**：模型无关（适用于黑盒LLM），评估器可替换，奖励函数设计（指数乘积+硬约束）有效平衡安全与质量，防止牺牲一维。
- **实验严谨**：使用多种不同类型的 LLM（未审查、微调、蒸馏）和丰富的不安全提示词汇，覆盖 12 类危害；进行了完整的性能-效率权衡和保留分析。
- **可复现性**：代码和数据集将公开发布，附录中包含详细状态、动作定义及超参数。
