## 摘要
视觉语言模型（VLM）在移动GUI导航方面取得了快速进展。本文系统研究了基于VLM的代理在这一领域的数据扩展、基准测试和推理。为了促进严格评估，我们引入了HyperTrack，这是一个大规模数据集，包含超过650个中国移动应用的16000多个实际任务，以及GUIEvalKit，一个开源工具包，用于在离线GUI导航任务上对VLM进行统一基准测试。使用HyperTrack，我们分析了训练数据规模对监督微调和基于强化学习的微调的影响。我们的结果表明，基于强化学习的微调始终优于监督微调，尤其是在域外设置中，突出了数据扩展与强化学习之间的协同作用。利用GUIEvalKit，我们进一步对最先进的VLM进行基准测试，并分析交互历史和推理能力如何影响任务完成。HyperTrack和GUIEvalKit共同为移动GUI导航任务中开发和评估VLM代理提供了一个全面的平台。

## Abstract
Vision-Language Models (VLMs) have shown rapid progress in mobile GUI navigation. This paper presents a systematic study of data scaling, benchmarking, and reasoning for VLM-based agents in this domain. To facilitate rigorous evaluation, we introduce HyperTrack, a large-scale dataset with over 16000 real-world tasks across more than 650 Chinese mobile applications, along with GUIEvalKit, an open-source toolkit for unified benchmarking of VLMs on offline GUI navigation tasks. Using HyperTrack, we analyze the effects of training data scale on both supervised and reinforcement-based finetuning. Our results show that reinforcement-based finetuning consistently outperforms supervised finetuning, particularly in out-of-domain settings, highlighting the synergy between data scaling and reinforcement learning. Leveraging GUIEvalKit, we further benchmark state-of-the-art (SOTA) VLMs and analyze how interaction history and reasoning capabilities influence task completion. Together, HyperTrack and GUIEvalKit provide a comprehensive platform for developing and evaluating VLM agents in mobile GUI navigation tasks.

## 6. 主要结论与发现
1. **数据缩放效应**：性能随训练数据量对数线性增长，RL在相同数据量下始终优于SFT，尤其在域外场景增益更大。这一趋势在不同骨干和奖励设计下一致。
2. **半在线评估（SOEval）**：能提供与在线评估（AndroidWorld）更强的相关性（Spearman ρ=0.771 vs 离线0.657），且通过引入on-policy上下文提升评估稳定性。
3. **推理模式的影响**：
   - 思考模式（Thinking）在PASS@1评估下往往表现更差，但扩大了决策多样性，在更宽容的评估（如PASS@8）下优势显现。
   - 推理提高了不稳定样本的稳定性，但降低了本已稳定的样本的稳定性，形成稳定-多样性权衡。
4. **推理-执行一致性**：接近一半的推理-执行不一致样本与执行失败相关，动作类型不匹配是主要错误类型（61.1%），动作目标不匹配占37.0%。
5. **专用GUI代理优于通用VLM**：UI-TARS、AgentCPM-GUI等在大多数基准上超过通用VLM，体现任务特定训练的价值。

- **全面的消融与敏感性分析**：对聚类参数、奖励设计、历史注入策略等进行详细稳健性检查。

## 8. 不足与局限
- **评估基于静态轨迹**：所有评估（离线、SOEval）仍基于预收集的参考轨迹，无法完全模拟真实交互的动态性（如错误恢复、非确定性）。SOEval只是近似，非替代。
- **数据集局限**：仅覆盖中文应用，可能限制跨语言和文化的泛化性。应用类型分布可能偏向某些类别（如生活、工具类）。
- **推理模式分析有限**：仅对GUI-Owl-7B做了详细推理/固定推理实验，其他模型（如Qwen3-VL）分析较浅。固定推理设置（固定推理文本）可能引入偏差。
- **聚类阈值偏好**：DBSCAN的epsilon=70已在敏感性分析中测试，但仍可能对密集UI元素区分不足（论文已承认会在紧凑布局中轻微低估多样性）。
- **计算开销**：获取决策分布需要大量重复采样（512次），实际部署成本高。
- **未涉及在线评估**：虽然引用了AndroidWorld，但自身未进行实机在线评估，无法完全验证SOEval的实用性。
- **隐私与安全**：数据集经过隐私过滤，但仍有残留风险，且未讨论恶意使用（如欺骗性UI攻击）的对抗鲁棒性。
