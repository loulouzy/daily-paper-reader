## 摘要
缺失模态会导致多模态推荐系统严重失效。在冷启动场景中，用户历史、物品文本和视觉证据经常缺失，而恰恰此时推荐质量最为关键。现有方法通过插补、特征传播或生成式重建来恢复缺失信号，但当幸存信号较弱时，这些策略可能注入无证据支持的信息。我们引入了元模态智能体（Meta-Modal Agent, MMA），一种基于大语言模型的候选池重排序器，将缺失性视为序列证据路由问题。MMA通过掩码模态片段上的平衡缺失性任务强化学习进行训练，并在两种变体上评估：MMA-Auto仅使用自动化文本、图像和图形工具；MMA-Interactive额外允许基于幸存模态的澄清问题，作为上限诊断。MMA在第一阶段检索器生成候选池后运行；它对候选进行评分，而非从全目录中检索物品。最终重排序融合MMA分数与在验证数据上选择的第一阶段检索分数。我们的评估围绕稳健缺失模态声明所需的四个证据检查进行组织：无先知单观测模态可用性（OOMA）鲁棒性、逐模态OOMA分解、固定池全目录重排序以及确定性路由器机制控制。MMA-Auto在目标正例OOMA NDCG@10上提升4.0%，在固定池全目录重排序NDCG@10上比最强非交互基线提升12.7%。RuleRouter-Fuse使用相同工具和融合规则但无学习策略更新，其性能劣于MMA-Auto，支持超越确定性工具融合的学习路由。MMA-Interactive在可以澄清时提供了4.1%的上限增益。

## Abstract
Missing modalities cause severe failures in multimodal recommender systems. User histories, item text, and visual evidence are frequently absent during cold-start scenarios, exactly when recommendation quality matters most. Existing approaches recover absent signals through imputation, feature propagation, or generative reconstruction, but these strategies can inject unsupported evidence when the surviving signals are weak. We introduce the Meta-Modal Agent (MMA), a large language model based candidate-pool reranker that treats missingness as a sequential evidence-routing problem. MMA is trained with balanced missingness-task reinforcement learning over masked-modality episodes and is evaluated in two variants: MMA-Auto, which uses only automated text, image, and graph tools, and MMA-Interactive, which additionally permits clarification questions grounded in surviving modalities as an upper-bound diagnostic. MMA operates after a first-stage retriever has produced a candidate pool; it scores those candidates rather than retrieving items from the full catalog. Final reranking fuses MMA scores with first-stage retrieval scores selected on validation data. Our evaluation is organized around four evidence checks required for a robust missing-modality claim: oracle-free one-observed-modality availability (OOMA) robustness, per-modality OOMA breakdowns, fixed-pool full-catalog reranking, and a deterministic-router mechanism control. MMA-Auto improves target-positive OOMA NDCG@10 by 4.0% and fixed-pool full-catalog reranking NDCG@10 by 12.7% over the strongest non-interactive baseline. RuleRouter-Fuse, which uses the same tools and fusion rule without learned policy updates, underperforms MMA-Auto, supporting learned routing beyond deterministic tool fusion. MMA-Interactive adds a 4.1% upper-bound gain when clarification is available.

- **整体含义**：论文将缺失模态问题从传统的“表示学习”重新定义为“序列证据路由”问题——即在一个部分可观测的环境中，智能体需要决定接下来查询哪个证据源（工具），并根据工具返回的 `Null`（缺失）信息调整后续动作，最终对候选池进行重排序。这种视角能够避免在弱信号下强行重建带来的噪声。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 提出 **Meta-Modal Agent (MMA)**，一种基于大语言模型（LLM）的候选池重排序器。
- 将冷启动缺失建模为一个部分可观测马尔可夫决策过程（POMDP），智能体通过与一组证据工具交互，逐步收集信息，最终输出候选物品的评分。
- 训练方式：使用 **平衡缺失性任务强化学习**（Balanced Missingness-Task Reinforcement Learning），在掩码模态的片段上进行训练，让智能体反复经历工具返回 `Null` 的情况并学会从中恢复。

### 关键技术细节

#### 模块化架构（图1）：
1. **模块1：片段接口与候选池**  
   - 每个片段共享一个候选池 $C_e$（大小 $B$），包含目标物品（隐藏，仅用于奖励计算）和 $B-1$ 个难负样本。  
   - 模态可用性由二进制掩码 $m \in \{0,1\}^{|M|}$ 决定（$M = \{ \text{text}, \text{img}, \text{beh} \}$），掩码不直接暴露，仅通过工具调用返回 `Null` 间接发现。

2. **模块2：证据路由智能体**  
   - 智能体（LLM）在每个时间步 $t$ 先输出推理链 $z_t$，再输出 JSON 格式的动作 $a_t$。  
   - **动作空间**：四种工具（三个自动化工具 + 一个可选交互工具）：
     - `Analyze_Text(item_id)`：返回标题、类别、描述；若 `$m_{\text{text}}=0$` 则返回 `Null`。
     - `Analyze_Image(item_id)`：返回 CLIP-ViT-L/14 的 top-5 语义标签；若 `$m_{\text{img}}=0$` 则返回 `Null`。
     - `Retrieve_Graph(user_id)`：返回 ANN 图检索的共购买邻居标题；若 `$m_{\text{beh}}=0$` 则返回 `Null`。
     - `Ask_User(modality_type, query)`：仅用于 MMA-Interactive 变体，属于理想化上限诊断。
   - 无效 JSON 或重复调用 `Null` 工具会触发惩罚。

3. **模块3：候选评分与融合**  
   - 终端动作 `Score_Candidates` 输出一个 JSON 映射（候选ID → 相关性分数 $s_i^{\text{MMA}}$）。  
   - MMA 分数经 min-max 归一化后与第一阶段检索分数融合：
     $$\hat{s}_i = \alpha s_i^{(0)} + (1-\alpha) s_i^{\text{MMA}}$$
     $\alpha$ 通过验证集网格搜索固定。

4. **模块4：平衡缺失性任务训练**  
   - 定义 7 个任务族 $\{\mathcal{T}_k\}_{k=1}^7$，对应不同掩码模式（如全观测、单模态缺失、仅单模态可用等）。  
   - 每个任务族分为支持集（80%用于策略更新）和查询集（20%用于诊断）。  
   - 训练时均匀采样任务族，避免策略被易学掩码主导。

- **奖励函数**：  
  最终排名奖励 = NDCG@10（基于融合分数排序）；步骤成本惩罚：工具调用 $-0.02$，用户询问 $-0.10$，无效动作 $-0.20$，重复调用 `Null` 工具 $-0.20$。

- **策略优化**：  
  使用 PPO 优化，广义优势估计（GAE），截断目标函数 $\epsilon=0.2$。仅训练 LoRA 适配器（$r=16, \alpha=32$）和 2 层价值头，基座 LLM 冻结。

- **两个变体**：
  - **MMA-Auto**：禁用 Ask_User，是主要的自动化部署版本。
  - **MMA-Interactive**：保留 Ask_User，仅作为上限诊断。

## 6. 主要结论与发现

1. **MMA-Auto 在 OOMA 场景下显著优于静态补全基线**：平均 NDCG@10 相对提升 4.0%（对比 DGMRec），在三个数据集上一致正向（+0.0064~+0.0068）。  
2. **逐模态优势一致**：MMA-Auto 在全部 9 个数据集×模态单元格上均胜出，尤其是在图像仅存场景下相对提升最大（+5.7%~+6.5%）。  
3. **全目录重排序验证**：在真实第一阶段检索产生的固定池上，NDCG@10 平均相对提升 12.7%，HR@10 提升 10.9%，但 Recall@100 不变，符合“重排序器不改进召回”的设定。  
4. **学习路由优于确定性工具融合**：RuleRouter-Fuse 的 NDCG@10（0.1578）低于 MMA-Auto（0.1711），且失败调用率更高（65.8% vs 47.0%）、轮数更多（3.8 vs 2.5）。  
5. **交互式澄清提供额外上限**：MMA-Interactive 在 MMA-Auto 基础上再提升 4.1%，相对 DGMRec 提升 8.3%，但该增益来自理想化的合成反馈。  
6. **消融证实各组件必要性**：零样本 ReAct 最差，SFT 提升明显，PPO 进一步改善，任务族均衡带来的增益较小但可诊断。  
7. **统计显著性**：所有三个数据集上配对 Wilcoxon 检验 p<0.05，Cliff's δ 在 0.15~0.19 之间，效应量中等。

## 8. 不足与局限

- **模型规模单一**：仅测试了 8B 参数的 Llama-3，未在更大模型（如 70B）上验证，无法观察规模缩放行为。

- **全目录重排序中的绝对 NDCG 很低**（0.02 左右），说明重排序器的改进空间有限，尤其在召回困难时——论文诚实指出这一点，但限制了实际应用吸引力。

- **RL 训练的超参数敏感**：仅报告一组默认超参数，未进行广泛的超参数搜索或敏感性分析（如奖励系数、批大小等）。
