## 摘要
通过强化学习（RL）训练大型多模态模型（LMM）原生调用视频处理工具（例如裁剪）已成为长视频理解的一个有前景的途径。然而，现有的原生RL方法按顺序调度工具调用（即每轮一次）：单个错误裁剪会传播错误而无法纠正，多轮工具调用会污染上下文，且推理成本随轮次线性增长。我们提出ParaVT，首个基于多智能体端到端RL训练的并行视频工具调用框架，在单轮中调度多个时间窗口裁剪，以实现更清晰的上下文和更好的容错性。然而，将标准RL应用于ParaVT暴露了一个我们称之为工具先验悖论的障碍：使工具探索成为可能的预训练工具先验，同时也破坏了冷启动的结构格式，并在温度采样下暴露了跳过工具奖励捷径。在弱先验LMM上的跨模型对比支持了这一观点：格式保持稳定，但RL引发零工具调用，表明先验强度是格式崩溃和工具探索的共同驱动因素。我们提出PARA-GRPO（基于解析锚定和比率门控的GRPO），通过两种互补机制增强标准RL：（i）仅在最容易崩溃的结构标记位置应用定向格式奖励，以及（ii）每个提示的帧预算随机化，创建训练提示，其中调用工具比跳过工具产生可测量的奖励信号。在六个长视频理解基准测试中，ParaVT相比Qwen3-VL基线平均提升7.9%，PARA-GRPO将训练时的格式合规性从0.13提升至0.64。随着工具能力在现代LMM中逐渐内化，RL必须与由此产生的先验协同工作，而ParaVT为智能体RL提供了一种通用方案。代码、数据和模型权重已公开。

## Abstract
Training large multimodal models (LMMs) via reinforcement learning (RL) to natively invoke video-processing tools (e.g., cropping) has become a promising route to long-video understanding. However, existing native-RL methods dispatch tool calls sequentially (i.e., one per turn): a single wrong crop propagates errors without peer correction, multi-turn tool calls corrupt context, and inference cost scales linearly with the number of turns. We introduce ParaVT, the first multi-agent end-to-end RL-trained framework for Parallel Video Tool calling, dispatching multiple time-window crops in a single turn for cleaner context and better fault tolerance. Yet applying standard RL to ParaVT reveals an obstacle we term the Tool Prior Paradox: the pretrained tool priors that enable tool exploration also destabilize cold-started structural format and expose the skip-tool reward shortcut under temperature sampling. A cross-model contrast on a weaker-prior LMM supports this claim: format stays stable but RL elicits zero tool calls, indicating that prior strength is the shared driver of both format collapse and tool exploration. We propose PARA-GRPO (Parseability-Anchored and Ratio-gAted GRPO), which augments standard RL with two complementary mechanisms: (i) a targeted format reward applied only at the structural-token positions most prone to collapse, and (ii) a per-prompt frame-budget randomization that creates training prompts where calling the tool yields a measurable reward signal over skipping it. Across six long-video understanding benchmarks, ParaVT improves over the Qwen3-VL baseline by +7.9% on average, with PARA-GRPO lifting training-time format compliance from 0.13 to 0.64. As tool capabilities become increasingly internalized in modern LMMs, RL must cooperate with the resulting priors, and ParaVT offers a general recipe for agentic RL. Code, data, and model weights are publicly available.

### 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：当前基于强化学习（RL）训练大语言模型（LMM）原生调用视频工具（如裁剪）的方法，多以**顺序方式**（每轮一次工具调用）进行。这种顺序范式存在三大缺陷：
  - 单次错误裁剪会传播而无法被同行纠正；
  - 多轮工具调用导致上下文污染；
  - 推理成本随调用次数线性增长。
- **整体含义**：本文旨在突破顺序限制，实现**并行工具调用**，同时解决由此带来的训练稳定性问题——即“工具先验悖论”（Tool Prior Paradox）：预训练形成的工具先验强度既是支撑工具探索的前提，又是导致格式崩溃和奖励捷径的根源。

### 2. 方法论：核心思想、关键技术细节

- **框架：ParaVT**  
  - 多智能体架构：一个主智能体在单轮中发出多个 `<tool_call>` 调用（对应不同时间窗口的裁剪），每个子智能体独立处理该窗口并返回文本摘要，而非重采样图像帧。主智能体汇总摘要后产出最终答案。
  - 优势：证据可交叉校验（错误可被纠正）；上下文增长可控（摘要而非图像帧）；推理延迟有界（子智能体并行执行）。

- **训练策略**  
  - 冷启动 SFT：在 Qwen3-VL-8B-Instruct 上进行多任务 SFT（97K 样本），让模型学会并行工具调用格式。
  - 然后进行 RL 微调（GRPO），并使用提出的 **PARA-GRPO** 算法解决工具先验悖论。

- **PARA-GRPO 关键技术**  
  - **探索锚定（Exploration Anchoring）**：针对格式崩溃（主要是闭合标签如 `</think>` 丢失）设计两种机制：
    - 约束生成：强制前缀 `<think>\n` 并奖励 `<answer>` 的存在；
    - 选择锚定：仅在最易崩塌的结构标记位置施加格式奖励（对正确闭合加分，对未闭合减分）。
  - **nFrames 门控（nFrames Gating）**：给每个提示随机分配概览帧数（4、8、16、32、64），使得部分提示无法从少量概览帧中直接得出答案，从而产生显著的“调用工具 vs 跳过工具”奖励差，避免 GRPO 群体的奖励归一化将差距抹零。
  - 两者结合：锚定确保生成可解析，门控则在这些可解析的生成序列上提供工具使用梯度。

### 6. 主要结论与发现

- ParaVT 在所有 6 个长视频基准上平均提升 **+7.9%**（相对于 Qwen3-VL-8B 基线），且在 7 个评估分割中赢了 6 个。
- PARA-GRPO 将训练时的格式合规率（`$f_τ$`）从 0.13 提升至 0.64，工具调用率（`$\kappa$`）稳定在 0.21，避免了 GRPO 下的格式崩溃和工具跳过捷径。
- 并行调度（推断时）优于顺序调度，且可通过一次性平行调用减少视频 token 重复编码（复杂度从 $O(K)$ 降为 $O(1)$）。
- 工具先验悖论通过跨模型对比（Qwen2.5-VL 弱先验 vs Qwen3-VL 强先验）得到验证：前者格式完美但无工具调用，后者探索工具但格式崩溃。

### 7. 优点

- **方法创新**：首次在视频工具调用中实现端到端 RL 并行化，并提出针对预训练先验的解决方法。
- **理论洞察**：识别并形式化了“工具先验悖论”，通过详尽的消融和跨模型实验支持该论点。
- **训练稳定性**：PARA-GRPO 的锚定和门控机制相互配合，既保证了格式可解析，又创造了必要的奖励差，解决了 RL 中的两个核心失败模式。
- **性能领先**：在多个基准上超越所有 7B/8B 级开源模型，甚至在某些基准上超过 GPT-4o。
- **开放共享**：代码、数据、模型权重均已公开，便于复现和后续研究。

### 8. 不足与局限

- **实验覆盖有限**：仅在 Qwen3-VL-8B 上验证了完整 pipeline，未推广到其他工具原生 LMM（如基于不同 tokenizer 或预训练目标的模型）。
- **跨模型证据强度**：工具先验悖论的支持主要来自 Qwen2.5-VL vs Qwen3-VL 的对比，缺少更精细的干预实验（如渐进削弱先验）。
- **仅测试 crop_video 工具**：未验证是否通用到其他工具（如文本检索、音频转录、场景图提取）。
- **RL 阶段贡献**：主要收益来自冷启动 SFT（头部收益），RL 带来的更多是部署时的稳定性和格式合规性，而不是大幅提升贪婪解码的评测分数。
- **可能的安全双刃剑**：并行工具调用提升了大规模监控等高风险场景的吞吐效率，需要部署时的透明度和同意框架。
- **负实验结果中的局限性**：某些尝试（如相奖励分段、任务感知系数）未见明显提升，说明算法设计具有领域特异性，仍需进一步通用化。
