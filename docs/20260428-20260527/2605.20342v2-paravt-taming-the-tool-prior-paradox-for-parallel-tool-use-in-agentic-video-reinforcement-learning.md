## 摘要
通过强化学习（RL）训练大型多模态模型（LMM）以原生调用视频处理工具（例如裁剪）已成为长视频理解的一条有前景的路径。然而，现有的原生RL方法按顺序调度工具调用（即每轮一个）：单个错误裁剪会在没有同伴纠正的情况下传播错误，多轮工具调用会污染上下文，且推理成本随轮数线性增长。我们提出ParaVT，这是首个用于并行视频工具调用的多智能体端到端RL训练框架，在单轮中调度多个时间窗口裁剪，以实现更清晰的上下文和更好的容错性。然而，将标准RL应用于ParaVT揭示了一个我们称之为工具先验悖论的障碍：使工具探索成为可能的预训练工具先验也会破坏冷启动的结构格式，并在温度采样下暴露出跳过工具奖励捷径。对较弱先验的LMM进行跨模型对比支持了这一观点：格式保持稳定，但RL引发零工具调用，表明先验强度是格式崩溃和工具探索的共同驱动因素。我们提出了PARA-GRPO（解析锚定与比率门控GRPO），通过两种互补机制增强标准RL：（i）仅在最容易崩溃的结构性标记位置应用有针对性的格式奖励，以及（ii）每个提示的帧预算随机化，创建训练提示，其中调用工具相对于跳过工具会产生可衡量的奖励信号。在六个长视频理解基准上，ParaVT平均比Qwen3-VL基线提升+7.9%，PARA-GRPO将训练时的格式合规性从0.13提升至0.64。随着工具能力在现代LMM中越来越内化，RL必须与由此产生的先验协同，而ParaVT为智能体RL提供了一个通用方案。代码、数据和模型权重已公开提供。

## Abstract
Training large multimodal models (LMMs) via reinforcement learning (RL) to natively invoke video-processing tools (e.g., cropping) has become a promising route to long-video understanding. However, existing native-RL methods dispatch tool calls sequentially (i.e., one per turn): a single wrong crop propagates errors without peer correction, multi-turn tool calls corrupt context, and inference cost scales linearly with the number of turns. We introduce ParaVT, the first multi-agent end-to-end RL-trained framework for Parallel Video Tool calling, dispatching multiple time-window crops in a single turn for cleaner context and better fault tolerance. Yet applying standard RL to ParaVT reveals an obstacle we term the Tool Prior Paradox: the pretrained tool priors that enable tool exploration also destabilize cold-started structural format and expose the skip-tool reward shortcut under temperature sampling. A cross-model contrast on a weaker-prior LMM supports this claim: format stays stable but RL elicits zero tool calls, indicating that prior strength is the shared driver of both format collapse and tool exploration. We propose PARA-GRPO (Parseability-Anchored and Ratio-gAted GRPO), which augments standard RL with two complementary mechanisms: (i) a targeted format reward applied only at the structural-token positions most prone to collapse, and (ii) a per-prompt frame-budget randomization that creates training prompts where calling the tool yields a measurable reward signal over skipping it. Across six long-video understanding benchmarks, ParaVT improves over the Qwen3-VL baseline by +7.9% on average, with PARA-GRPO lifting training-time format compliance from 0.13 to 0.64. As tool capabilities become increasingly internalized in modern LMMs, RL must cooperate with the resulting priors, and ParaVT offers a general recipe for agentic RL. Code, data, and model weights are publicly available.

## 1. 核心问题与整体含义（研究动机和背景）

- **背景**：长视频理解任务中，大型多模态模型（LMM）通过强化学习（RL）后训练来原生调用视频处理工具（如裁剪）已成为趋势。现有方法（如LongVT、Conan等）按顺序调度工具调用——每轮一个工具调用，多轮累积导致三个核心问题：
  - **错误传播**：单个错误的裁剪无法得到同伴纠正，错误沿串行链传播。
  - **上下文污染**：多轮调用的输出（尤其是视觉token）不断累加到上下文中，增加噪声和计算开销。
  - **推理成本线性增长**：每轮都需要重新处理上下文，总耗时随工具调用次数线性增加。
- **核心悖论**：作者发现将标准RL（如GRPO）应用于并行工具框架时，出现**工具先验悖论**：预训练获得的工具先验（如Qwen3-VL的`<tool_code>`模式）虽然在冷启动时能促进工具探索，但同时在温度采样下会：
  - 破坏SFT习得的结构格式（格式脆弱性：`<tool_call>`退化回`<tool_code>`，缺失闭合标签）。
  - 暴露出跳过工具的奖励捷径（工具必要性差距：当概览帧足以回答问题时，调用工具与跳过工具的奖励差异几乎为零，导致RL策略迅速收敛到跳过工具）。
- **研究动机**：驯服这一悖论，使RL与模型内化的工具先验协同，实现稳定、高效的并行工具调用。

## 2. 方法论：核心思想、关键技术细节

- **ParaVT框架**（并行视频工具调用）：
  - **多智能体架构**：一个主智能体在单轮中发出多个`<tool_call>`，每个调用指定一个不重叠的时间窗口。这些调用被分发给多个共享权重的子智能体并行执行。
  - **子智能体**：每个子智能体只处理其窗口，返回文本摘要（而非重采样帧），主智能体收集所有摘要后聚合证据给出最终答案。
  - **优势**：同伴可纠正错误（多数投票）、上下文增长可控（文本摘要轻量）、推理延迟不随工具数量线性增长（并行执行）。

- **PARA-GRPO**（解析锚定与比率门控GRPO）：针对工具先验悖论的两个失败模式，提出两种互补机制：
  1. **探索锚定**：解决格式脆弱性。
     - **约束生成**：强制响应以`<think>\n`开头，并在格式奖励中对`<answer>`闭合给予信用。
     - **选择性锚定奖励**：仅在结构边界标记（闭合`</think>`、`<think>`→`</think>`→`<answer>`流程）处施加奖励。公式（1）：
       - `+α`若`</think>`正确闭合；
       - `+β`若完整流程保留；
       - `-γ`若`<think>`打开但未闭合。
     - 锚定奖励与准确率梯度相加，不竞争。
  2. **nFrames门控**：解决工具必要性差距。
     - 每个GRPO组（G=8个轨迹）中，概览帧数从集合{4,8,16,32,64}中随机均匀选取。
     - 低帧数时，部分视觉证据丢失，调用工具重新获得证据可显著提高奖励，从而创造非零的工具/跳过工具优势比；高帧数保留容易模式，避免过度调用。
     - 作用前提：仅当轨迹可解析时，门控梯度才能归因于工具使用行为，因此探索锚定必须先生效。

- **整体奖励函数**：`R = R_acc + λ_fmt·R_fmt + R_tool`，其中`R_fmt`包含基础格式奖励和锚定奖励`R_anchor`，`R_tool`是解析良好的`<tool_call>`的小额奖励。

- **训练流程**：
  - **SFT冷启动**：在97K多任务数据集上微调（包括通用视频QA、并行工具轨迹、时序定位等），选择格式稳定性好的早期检查点作为RL初始化。
  - **RL**：在4.4K样本上使用PARA-GRPO训练，组大小G=8，温度τ=0.7，锚定权重λ_anchor=0.5。

## 6. 主要结论与发现

- ParaVT（并行工具调用）在所有六个长视频理解基准上显著优于现有开源模型，平均提升+7.9%相对Qwen3-VL基线。
- PARAGRPO有效解决了工具先验悖论：训练格式合规性从0.13提升至0.64，工具调用率稳定在适度水平（0.21次/轨迹），避免过度或不足调用。
- 并行模式在相同训练检查点上优于顺序模式，说明并行调度本身带来精度提升。
- 工具先验悖论是真实且普遍的现象：强先验模型（Qwen3-VL）在标准GRPO下格式崩溃，弱先验模型（Qwen2.5-VL）格式完美但零工具调用。
- PARA-GRPO的两个组件（探索锚定和帧门控）缺一不可，且顺序重要：必须先用锚定确保格式可解析，帧门控才能有效创造工具使用梯度。

## 7. 优点

- **方法创新性**：首次提出端到端RL训练并行视频工具调用框架，识别并命名了工具先验悖论，提供了系统性解决方案。
- **设计巧妙性**：探索锚定仅在结构边界施加奖励，不与准确率梯度竞争；帧门控通过随机化概览帧数自然创造工具必要性的梯度，无需硬阈值。

- **开源友好**：代码、数据、权重公开，便于复现和拓展。
