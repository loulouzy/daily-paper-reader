## 摘要
当人类描述视觉场景时，他们并不会均匀地处理整幅图像；相反，他们会选择性地注视与其意图描述相关的区域。相比之下，当前的多模态大语言模型（MLLMs）在每个生成步骤中都关注所有视觉标记，导致注意力分散和不必要的计算开销。在这项工作中，我们引入了注视注意力（Gaze Attention），一种新颖的机制，使MLLMs能够在生成过程中选择性地关注与任务相关的视觉区域。具体来说，我们将视觉嵌入（以键值缓存形式存储）在空间上分组为紧凑的注视区域，每个区域由一个轻量级描述符表示。在每个解码步骤中，模型动态选择最相关的区域并仅关注这些区域，从而减少冗余计算并增强聚焦。为了缓解局部注意力导致的全局上下文丢失，我们进一步提出了可学习的上下文标记，将其附加到每个图像或帧上，使模型能够保持整体的视觉感知。在图像和视频理解基准上的大量实验表明，注视注意力在匹配或超越密集注意力基线的同时，在注意力计算中使用的视觉键值条目数量减少了高达90%。

## Abstract
When humans describe a visual scene, they do not process the entire image uniformly; instead, they selectively fixate on regions relevant to their intended description. In contrast, current multimodal large language models (MLLMs) attend to all visual tokens at each generation step, leading to diluted focus and unnecessary computational overhead. In this work, we introduce Gaze Attention, a novel mechanism that enables MLLMs to selectively attend to task-relevant visual regions during generation. Specifically, we spatially group visual embeddings-stored as key-value caches-into compact gaze regions, each represented by a lightweight descriptor. At each decoding step, the model dynamically selects the most relevant regions and restricts attention to them, reducing redundant computation while enhancing focus. To mitigate the loss of global context caused by localized attention, we further propose learnable context tokens appended to each image or frame, allowing the model to maintain holistic visual awareness. Extensive experiments on image and video understanding benchmarks demonstrate that Gaze Attention matches or surpasses dense-attention baselines, while using up to 90% fewer visual KV entries in the attention computation.

## 1. 核心问题与整体含义（研究动机和背景）
- **问题**：当前多模态大语言模型（MLLMs）在生成每个 token 时，都会**不加区分地关注所有视觉 token**（包括无关区域），导致注意力分散和计算冗余。这与人类选择性注视的视觉机制形成鲜明对比。
- **意义**：本文旨在为 MLLMs 引入类似人类的目光选择性注意力，使模型在生成过程中**动态聚焦于任务相关视觉区域**，从而提升生成质量并降低计算开销。

## 2. 论文提出的方法论
### 核心思想
- 将视觉 KV 缓存（key-value caches）按空间局部性**分组为紧凑的注视区域**（gaze regions），每个区域由一个轻量级描述符（descriptor）表示。
- 在每一步解码时，只允许查询（query）**选择并关注少数最相关的区域**，实现稀疏局部注意力。
- 为弥补局部注意力丢失的全局场景信息，引入**可学习的上下文 token**（per image/frame），保持整体视觉感知。

### 关键技术细节
- **视觉区域分组**：将视觉键（key）向量重塑为原始空间网格，划分为 \( r_h \times r_w \times r_t \) 个 token 的区域（默认 6×6×1）。
- **区域描述符**：对区域内的所有 key 向量**取均值池化**，得到紧凑的表示 \( d_u^g \)。
- **视觉路由**：解码时，查询 \( q_j \) 与所有区域描述符计算点积相似度，选择 TopK 区域 (\( G_j \))，仅对这些区域的 KV 对进行注意力计算。
- **可学习上下文 token**：每个视觉单元附加一组（默认 4 个）可学习 token，这些 token 在 LLM 层中传播后产生全局上下文 KV 缓存 \( k_u^C \)，对所有生成步骤一直可见。
- **注意力公式**：结合文本 KV、全局上下文 KV 和选定区域的 KV：  
  \( \text{GazeAttn}(q_j) = \text{Softmax}(\frac{q_j K_{\text{Gaze}}(q_j)^\top}{\sqrt{d}}) V_{\text{Gaze}}(q_j) \)。
- **训练稳定性**：采用**渐进式 TopK 调度**，从选择所有区域（100%）逐步线形减少到 10%，避免初期路由不稳定。

## 6. 论文的主要结论与发现
- **性能匹配或超越密集注意力**：在 13 个图像和 6 个视频基准上，Gaze Attention 的平均得分与密集注意力基线相当或更好（如 Cambrian-4B+GA 72+4 KV 在图像平均分 68.1 vs 基线 68.6，几乎持平）。
- **计算效率大幅提升**：注意力 FLOPs 降低约 **81%**，KV 缓存 GPU 内存降低约 **79%**，视觉 KV 条目减少 **90%**（例如从 576 降至 72+4）。
- **可解释的注视模式**：可视化显示注意力图从“弥漫”变为“集中”，且在不同生成步骤中自动切换关注区域，模仿人类注视（无需显式注视监督）。
- **优于现有 KV 逐出方法**：InfiniPot-V 和 HERMES 在同等 KV 预算下性能显著下降（如 50 KV 时平均分从 71.0 降至 61.4），而 Gaze Attention 保持稳健。

## 7. 优点
- **动态适应性**：每步解码重新选择区域，避免了固定重要性评估导致的信息丢失；可处理多区域描述的任务。
- **全局与局部兼顾**：通过可学习上下文 token 保留全局感知，弥补局部注意力的不足。
- **训练简单稳定**：仅靠下一个 token 预测损失，无监督信号；渐进式 TopK 调度有效解决稀疏注意力训练的困难。
- **兼容性好**：可与输入级视觉 token 压缩（如卷积下采样）结合，进一步减少 KV 缓存。
- **内存优化实用**：采用 KV 缓存卸载策略（CPU/SSD + GPU 按需加载），在大幅减少 GPU 内存的同时保持推理可行性。

## 8. 不足与局限
- **固定区域大小**：6×6 的固定网格可能切割语义边界（如物体跨区域），导致信息碎片化；内容自适应分割需要进一步探索。
- **描述符简单**：均值池化平等对待区域内所有 token，可能弱化显著物体信息；更复杂的加权（如基于熵的重要性）可在额外计算成本下提升路由质量。
- **缺乏显式注视监督**：虽然模型自主学会了类似人的注视，但若能在训练中引入语言-区域对齐信号（如基于句子解析和开放词汇检测器生成伪标签），可进一步提升定位精度。
- **推理加速幅度有限**：由于稀疏注意力算子的工程实现挑战（无法利用 FlashAttention 等高度优化内核），Gaze Attention 的实际 wall-clock 时间仅为密集基线的 2.6 倍，而 InfiniPot-V 和 HERMES 分别为 3.5 倍和 3.6 倍。该问题属于共享的工程瓶颈，需要后续联合稀疏注意力算子予以解决。
- **架构迁移性未验证**：实验仅基于 Qwen+SigLIP2，理论方法应适用于其他 MLLMs，但未在更多种架构（如 LLaVA、InternVL）上复现验证。
