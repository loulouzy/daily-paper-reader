## 摘要
记忆对于大型视觉-语言模型（LVLMs）处理长程多模态交互至关重要，提供这一能力的方法方向包括长上下文LVLMs和记忆增强型智能体。然而，现有基准测试尚未在真正需要多模态证据的问题上对两者进行系统比较。为填补这一空白，我们提出了MEMLENS，一个针对多模态多会话会话中记忆的综合基准测试，包含789个问题，涵盖五种记忆能力（信息提取、多会话推理、时间推理、知识更新和拒绝回答），在跨模态标记计数方案下采用四种标准上下文长度（32K-256K token）。图像消融研究证实，解决MEMLENS需要视觉证据：移除证据图像后，在80.4%证据包含图像的问题上，两个前沿LVLMs的准确率降至2%以下。评估了27个LVLMs和7个记忆增强型智能体后，我们发现长上下文LVLMs通过直接视觉定位实现了高短上下文准确率，但随着对话增长而退化，而记忆智能体长度稳定但在存储时压缩下失去视觉保真度。多会话推理将大多数系统限制在30%以下，单独一种方法都无法解决该任务。这些结果激励了结合长上下文注意力与结构化多模态检索的混合架构。我们的代码可在https://github.com/xrenaf/MEMLENS获取。

## Abstract
Memory is essential for large vision-language models (LVLMs) to handle long, multimodal interactions, with two method directions providing this capability: long-context LVLMs and memory-augmented agents. However, no existing benchmark conducts a systematic comparison of the two on questions that genuinely require multimodal evidence. To close this gap, we introduce MEMLENS, a comprehensive benchmark for memory in multimodal multi-session conversations, comprising 789 questions across five memory abilities (information extraction, multi-session reasoning, temporal reasoning, knowledge update, and answer refusal) at four standard context lengths (32K-256K tokens) under a cross-modal token-counting scheme. An image-ablation study confirms that solving MEMLENS requires visual evidence: removing evidence images drops two frontier LVLMs below 2% accuracy on the 80.4% of questions whose evidence includes images. Evaluating 27 LVLMs and 7 memory-augmented agents, we find that long-context LVLMs achieve high short-context accuracy through direct visual grounding but degrade as conversations grow, whereas memory agents are length-stable but lose visual fidelity under storage-time compression. Multi-session reasoning caps most systems below 30%, and neither approach alone solves the task. These results motivate hybrid architectures that combine long-context attention with structured multimodal retrieval. Our code is available at https://github.com/xrenaf/MEMLENS.

### 6. 论文的主要结论与发现

1.  **两种方法互补但各有致命短板**：
    *   **长上下文LVLMs**：在短上下文中通过直接视觉定位取得高准确率，但随着对话长度增加，在检索视觉证据和进行多会话推理时性能显著下降。
    *   **记忆增强型智能体**：性能对对话长度相对稳定（长度不变性），但在将视觉信息压缩存储到记忆库时丢失了大量精细视觉特征，导致视觉相关任务准确率低。此外，为了记忆管理而进行的微调（如RL）削弱了模型的“拒绝回答”能力。

2.  **能力不同源**：五种记忆能力之间的相关性较低，说明它们衡量的是不同的能力维度。强“信息提取”不代表强“多会话推理”。

3.  **最大瓶颈**：**多会话推理（MSR）** 是所有模型的共同瓶颈，最高准确率不到30%。Oracle诊断表明，这个瓶颈主要在**信息检索**（无法在长序列中准确定位所有相关对话），而非推理能力本身。

4.  **未来方向**：单一的模型扩展或存储优化都无法解决问题，论文指出未来发展应侧重于**混合架构**，即将长上下文注意力与结构化多模态检索相结合。

2.  **跨模态依赖设计**：通过“实体抽象”等创新方法，确保问题必须结合图像和文本才能回答，解决了以往基准中视觉信息冗余的问题。
3.  **长度可控性**：通过动态调整干扰会话，生成了四种标准长度的上下文，使得对不同架构的比较更加公平。
4.  **深入的分析**：论文不仅仅报告了总分数，还通过错误分解、能力相关性分析、Oracle诊断和智能体管道拆解等多种分析手段，深入揭示了模型失败的根本原因。
5.  **全面的智能体评估**：不仅评估了LVLMs，还评估了多种设计和后训练方式的记忆体，并分析了其失败的根源（如检索失败 vs. 推理失败），为未来设计提供了有价值的指导。
6.  **严谨的评估流程**：使用LLM-as-Judge评估，并通过与独立LLM和人工评估进行交叉验证，保证了评估结果的信服力。

### 8. 不足与局限

1.  **合成对话**：所有会话（包括证据和干扰项）均由LLM生成，虽然经过人工审核和干扰性验证，但可能与真实的人机交互数据存在分布差异。数据集尚不能完全代表真实用户-助手交互的分布。
2.  **数据集规模**：最终的基准仅包含789个问题，属于中等规模。更多的问题可以提高测试的稳定性和区分度。
3.  **评估偏差**：用于打分的LLM（Qwen3-VL-235B）与一些被评估的模型（Qwen系列）属于同一家族，存在潜在的“家族亲缘性”偏差。虽然论文进行了交叉验证证明影响不大，但该风险并未完全消除。
4.  **智能体适配器影响**：不同智能体处理视觉输入的适配器不同（如图像标题 vs. 原始像素），这本身就是一个混淆因素。性能差异可能部分来自于适配器，而非记忆系统核心架构。
5.  **静态评估**：当前评估是在已冻结的对话历史基础上进行的离线测试。与实际应用中智能体在线、增量式地读写记忆的场景（流式评估）仍有差距，未能完全模拟“写入”和“遗忘”的动态过程。
